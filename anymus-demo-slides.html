Differentiator D3 — Pre-flight Health Check

The Gate That Protects Everything

Before any test runs — API or UI — a pre-flight health check pings all critical endpoints. Infrastructure failure won't masquerade as test failure.

$ python scripts/health_check.py --base-url http://inventree.localhost
ANYMUS — Pre-flight Health Check
Target: http://inventree.localhost
Endpoint                  Status  Latency
/api/                    200    40ms   ✓ UP
/api/part/               401    6ms    ✓ UP
/api/part/category/      401    5ms    ✓ UP
/api/part/category/tree/ 401    6ms    ✓ UP
/api/part/category/param/ 401    5ms    ✓ UP
/api/part/test-template/ 401    5ms    ✓ UP
Result: ALL SYSTEMS GO ✓ (6/6 healthy)
How It Works
  • 1. Pings 6 endpoints with 5s timeout each
  • 2. 200 or 401 = healthy (server responds)
  • 3. 500, 404, or refused = unhealthy
  • 4. Outputs colored table to console
  • 5. Writes health_report.json for CI artifacts
  • 6. Exit code 0 (all up) or 1 (any down)
Pytest Integration

test_meta_health.py runs first (filename prefix). The session-scoped api_health_gate fixture in conftest.py calls pytest.skip() on the entire session if any endpoint is down — protecting both API and UI test suites.

Differentiator D2 — Mock API Mode

Testing Never Stops

When InvenTree is unreachable, MOCK_MODE=true activates. The responses library intercepts all HTTP calls and replays recorded JSON fixtures — same business logic, zero infrastructure.

Mock Fixtures — automation/api/mocks/
part_list.json GET /api/part/ → 3 parts, all fields
part_detail.json GET /api/part/1/ → single part, complete
part_create_201.json POST /api/part/ → success response
part_create_400.json POST /api/part/ → validation error
category_list.json GET /api/part/category/ → hierarchy
$ MOCK_MODE=true pytest tests/test_part_mock.py -v
10 passed in 0.24s
✓ Zero network calls — completely offline
What Mock Tests Cover
  • ✓ Part list returns count + results
  • ✓ Part fields present and typed correctly
  • ✓ Assembly flag in fixture is correct
  • ✓ Part detail has notes field
  • ✓ Create success returns 201
  • ✓ Create with missing name returns 400
  • ✓ Category list has hierarchy fields
  • ✓ Category level reflected in pathstring
Key principle: Mock tests exercise the same business assertions as live tests — only the transport layer differs. This guarantees that switching MOCK_MODE on/off produces consistent results.
